例）

ビジネスでAIを活用するという事において、ビジネスデータをAIと接続するという基盤が重要
96%の企業はデータ基盤を安定させている事で本番環境に実装している

### 課題

- データのサイロ化

様々なデータが分離・独立してしまっている
このサイロ化しているデータを繋げる事が重要になる
↑
重要になる事が、セキュリティに関する事になる

Snowflakeの企業理念 => 世界のデータを活用可能に
クラウドでデータを貯める => データを整備してみんなが使う事ができるようにする

データ戦略が鍵になる

### 3つの障壁になりうるポイント 

複雑・サイロ化・セキュリティ　が大きな課題

複雑 => Easy
サイロ化 => Connect
セキュリティ => Trust

Snowflakeの全体像
クラウドでこれらを管理して、簡単に接続して、信頼性高くそれらのデータを使う事ができるようになる

#### Easy

- プロダクト例
    - Cortex code => コード生成に特化したAIエージェント
    - Snowflake Intelligence => チャット型のAIエージェント
    - PythonのStreamlitを使ったダッシュボードを作成する事などが可能

- Openflow
    - Connecterで定義されたツール
        - Google Driveなど
- Snowflake Marletplace
    - データプロバイダーとの契約によって任意のデータを取得する事ができる
        - ニュースの情報
        - 金融
        - マーケ
        - etc...
    - 無料も有料も複数ある
    - データの移動を行わずプロバイダーが持っているデータを直接取り込む

- Snowflake Intelligence
    - チャット形式で必要な分析をしてくれるapp

#### Connect

- データ形式
    - 構造化データ
        - テーブル形式と呼ばれたりする
            - 細かくかっちり決まっている
    - 半構造化データ
        - データの意味が付いているデータ
            - JSON
            - XML
                - 中間データというイメージ
                    - データ形式
    - 非構造化データ
        - 画像データ
        - 音声データ

#### Trust

セキュリティや信頼性の部分

事業の始まる => DBのようなものが始まり

- ユーザーに応じて権限の設定を行うとともに、LLMに対しても権限の設定を行う事ができる
    - アクセス制御
        - 行アクセスポリシー
        - 列レベルポリシー
        - データ本体は見れないが、計算結果のみは取得する事ができる
            - など

- SnowflakeのLLM
    - 機械学習にデータは使わない
        - Snowflakeの中でLLMが動いているという担保

- 出力の根拠や理由等を出してくれる
    - 可観測性
    - 評価
    - 説明可能性
    - ガードレール

- 対応するクラウドは自分で選択可能
    - AWSやAzure,GCP
        - リージョンも自身で設定する事が可能
