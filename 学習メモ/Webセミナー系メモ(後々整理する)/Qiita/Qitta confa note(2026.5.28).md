# 価値の再定義 - AI時代における提案・共創の新しいルール

KPIは何を測っていないか？
↑
KPIを達成するために普段様々な事を行っている

ソフトウェアの手の内か => 内製化

[価値の再定義]
AI時代において価値の転換が起きている
- G-D Logic
    - 作られた瞬間に価値が存在している => Value in Exchange => 購入された瞬間に価値が映る
- G-D Logic
    - 価値は使う瞬間に生まれる => 文脈に存在する => 作る側と使う側が同時に存在する

価値は、使われた瞬間に文脈の中で生まれる
- 従来の構造
    - 作る人 => 使う人 => 作る人
- AI後
    - 作る人 <> AI <> 使う人
        - AIが作るし、AIが使う
            - AIがサービスを利用する事があり得る

共創はどこで生まれるか？
- 基盤 => ナレッジ
    - しっかりとした文脈が存在しないと様々な方向へ価値が逃げてしまう

組織のナレッジ
- フロー型
    - Slackの会話
    - ミーティングの議論
    - 廊下での立ち話
- ストック型
    - SSoT
    - ドキュメント
    - コード
    - ADR

- 今後に必要な事
    - フロー型とストック型の橋渡しをする事が重要
        - AIに渡す事が出来る情報はストック型の情報のみになってしまう
            - 新しい人がオンボーディングした時にもストック型の情報は重要になる

- AI時代において重要な指標
    - プロダクトの4階層
        - Core

            - ここを境に人間とAIのチェンジが起きる

            - 色々な面で言われるが、なぜ作るのか？が重要になる

- 作る => 創る
    - 形にして出力するようなイメージ

- 人間に残る役割
    - 目的地を定める
    - ガードレールになる
