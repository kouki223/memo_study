FargateというAWSのコンテナ単位で処理を起動する事が出来るサービスを使って処理の定期実行を行なっている
Lambda関数で動く自動取り込みとは使っているサービスが違っている

[FargateとLambdaの違い]
- 違い
    - 実行単位
        - Fargate
            - コンテナ単位
        - Lambda
            - 関数単位
    - 実行可能時間
        - Fargate
            - 無制限
        - Lambda
            - 最大15分
    - 起動までの時間
        - Fargate
            - 数十秒
        - Lambda
            - 数秒
    - 課金単位
        - Fargate
            -  vCPU/メモリ × 秒
        - Lambda
            - 呼び出し回数 × 実行時間
    - スケーリング
        - Fargate
            - ECS/EKSで制御
        - Lambda
            - 自動（同時実行数に上限あり）

- 利点
    - Fargateは実行時間に制限が存在しない。
    - そのため、長時間実行することが必要なサービス等に向いている

- コスト的な観点
    - Lambda は呼び出しがない間は課金ゼロなので、アクセス頻度が低い用途はコスト優位
    - Fargate は起動している時間分課金されるため、常時稼働させる場合は計算が必要

├── Dockerfile

    - 環境変数等の設定ファイル
└── template.yaml
    - AWS(Fargate)にデプロイを行う際の各種SAMテンプレート
