# オブジェクト指向プログラミング(OOP:object-oriented programming)について

参考リンク：[MDN オブジェクト指向プログラミング](https://developer.mozilla.org/ja/docs/Learn_web_development/Extensions/Advanced_JavaScript_objects/Object-oriented_programming)

### オブジェクト指向プログラミングとは？(以降、オブジェクト指向プログラミングはOOPと表記)
- 多くの言語のプログラミングパラダイムであると言える
    - クラスとインスタンス、継承、カプセル化と呼ばれる3つの概念を含む
        - オブジェクトを基本単位としてプログラムする指向
            - データとその操作をまとめたもの

### 前提知識
    ⚪︎オブジェクトについて
    ⚪︎ドット記法
    ⚪︎ブランケット記法
    ⚪︎thisについて
    ⚪︎コンストラクター(newキーワード)、関数について
    ⚪︎オブジェクトによって受ける事ができるWebAPIや組み込みメソッドの使用について(共通化)
    ⚪︎組み込みAPI毎の仕様に関する注意(全てのAPIや組み込みメソッドがインスタンスを生成するわけではないという事)
    ⚪︎オブジェクトによってパッケージ化した状態で関数や変数を保持する事で有害(クラッシュする可能性)な影響から守られる

## オブジェクト指向の概念整理

- Value：オブジェクト指向の目指す事
    - システムをオブジェクトの集合体とする事でモデル化する事
        - オブジェクトの内部には、関数とデータ(データと処理：プログラムの本質)が格納されている
            - オブジェクト単位で関係性や振る舞いを表す事でシステム(処理を抽象化したもの：要はモデル化)を表現する
                - Webアプリケーションにおいては、現実を抽象化したものがWebシステムと呼ばれる

- 抽象化(クラスとインスタンス)
    - is-a
        - 汎用化（継承）
    - part-of
        - 集約関係
            - 〜からなる
    - 階層構造化する
        - オブジェクト指向でモデル化したい問題が学校だった場合
            - 学校には教授というオブジェクトが存在する事を表現できる
                - 教授というオブジェクトはプロパティとして、論文の採点、自己紹介などをする事ができると考える事ができる
                    - つまり、大きな学校というシステムにおける教授というクラスを定義する事ができる
        ```javascript
        class Professor
            properties
                name
                teaches
            methods
                grade(paper)
                introduceSelf()
        ```
        - 上のクラス定義を言語化すると
            - 2つのデータプロパティ
                - name
                - teaches
            - 2つのメソッド
                - grade() 
                    - 論文の採点
                - introduceSelf() 
                    - 自己紹介を行うメソッド
                        - となる
        - クラスを定義するという事は振る舞いを決定しているのではなく、そのインスタンスがどのような構造をしているかという内部構造を示しているという事になる
            - インスタンスというクラスの内部構造を表す初期値が存在している
                - コンストラクターと呼ばれる特別な関数を使う事も可能(newキーワードなどが存在する)
        ```javascript
        class Professor
            properties
                name
                teaches
            constructor
                Professor(name, teaches)
            methods
                grade(paper)
                introduceSelf()
        ```
        - ポリモーフィズム　＝＞　多様性
            - 異なるクラスのオブジェクトが同じインターフェースを共有する。
                - 共有した上で自身に適切な形で実装する
                    - 人間
                        - 走る
                            - 人によって
                                - 早くor 遅く　など
                                    - 同じインターフェースを適切に実装するという事

- 継承
    - 親クラスの機能を子クラスに引き継ぐ
        - Productクラス　＝＞ Foodクラス
            - メソッド
                - 合計金額を計算する
                - 合計個数を計算する
            - プロパティ
                - 食べ物の名前
                - 食べ物の値段
    - 同階層に存在する共通する項目を親クラスに抽象化する
        - Productクラス
            - Foodクラス
            - Drinkクラス
                - これらのには
                    - 上記で示したようなメソッドとプロパティが想像される
                        - 追加でここに追加されるプロパティ
                            - Foodクラス
                                - プロパティ
                                    - 容量(g換算)
                                - メソッド
                                    - 総重量の計算
                            - Drinkクラス
                                - プロパティ
                                    - volume (ml)
                                - メソッド
                                    - 総volの計算
        ```javascript
        class Product
            properties
                name
                price
                total_cal
            method
                calcurate_price(count)
            constructor
                Product(name, price, total_cal)
        
        class Food : extends Product
            properties
                glam
            method
                calucurate_total_glam(gram)
            
        class Drink : extends Product
            properties
                volume
            method
                calucurate_total_volume(ml)
        ```
        - このように定義する事で、抽象化した時に共通して示されるものに関しては親クラスで定義する
            - 子クラスでは、親クラスのスーパーセットとして定義する事でポリモーフィズムを実現している
                - 親クラスと子クラスで同じメソッドが存在する場合には子クラスのメソッドが上書きする

- カプセル化
    - オブジェクトは内部構造を非公開にします。
        - 公開IF
        - 非公開
            - 2つの部分に切り分ける事がカプセル化と呼ばれる
    - カプセル化とは単純に内部構造を非公開でプライベートにするという事ではなく、適切な外部IFを作成する事になる
        - 関数への切り出しなども含まれる

    - データ構造
        - データの操作方法
            - 直接的にデータの内容を書き換える事を禁止
                - データに対しては操作対象を指定した上でデータの操作を行う
                    - カプセル化していないデータで変更の指示を与える
                - 外部からの直接的な操作を禁止する
                    - 操作は禁止するが公開は可能になる
