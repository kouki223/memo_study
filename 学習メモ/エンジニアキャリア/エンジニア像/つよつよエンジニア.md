# T字型の人材になる(つよつよエンジニアとはどのようなエンジニアか？)
- 1つの目安としてエンジニア歴6年目でつよつよエンジニアに到達する事が出来れば良い成長角度だろう
    - 技術(ソフト・ハード含めた)スタック
        - T字型人材
            - 特定の技術領域において深い専門性を持っている * 周辺領域、関連技術含めた幅広い技術の知識
        - トラブルシューティング
            - 影響範囲の迅速な特定や原因の精密な切り分け、そして実行可能な解決策の提示までを一気通貫してできる
        - ソフトスキル
            - 他チームや顧客からも技術的な相談や協力を求められるような存在になる事で円滑なコミュニケーションが可能

## 7つの習慣 => 日々の積み重ねが重要になる
    - 技術情報のキャッチアップ
        - Google Discover
        - X
        - YouTube
        - Qiita
            - 全ての情報源に毎日というわけではないが、上位のトレンド情報において、知らない単語や情報があった場合には、調べるなどという対応ができるようにする事でT字型の人材への道につながる
    
    - 普段、何気なく使っているサービスの内部構造に興味を持つ
        - 普段使っている数多くのサービスにおいて、内部構造、アーキテクチャなどに興味を持ち、公開されているものがあれば学習する
            - 興味を持ったらすぐに調べるという習慣がある事が重要

    - 自分のチーム以外かつ、同じプロダクトを開発しているチームの成果物を見る
        - 設計書、パラメーターシート、コードなどを積極的に読む事で多くの学びを得る事ができる、かつ、自分が関わっているプロダクトであるため、深い理解ができる

    - ドメインを深く理解する
        - 特定のドメインにおける業界知識や、専門用語の理解などは重要
            - ビジネスサイドで当たり前に使われている単語などを理解した上で円滑なコミュニケーションが可能になる

    - アウトプットする習慣
        - アウトプットする事自体に意義があるが、その過程も重要
            - 知識整理、質疑応答、フィードバックなどによってさらに深く、整理された知識へ

    - ソフトスキル
        - 相手に分かりやすく説明する能力
        - 相手の意図を正確に理解する能力
        - 相手と建設的な議論を行う能力
        - 相手に敬意を払って会話する能力　

    - 習慣によってチャンスを掴み取る
        - 習慣を続ける事でチャレンジングなものを経験する事ができる
            - チャレンジングなものを経験する事でさらに技術、経験が昇華する

    - フルスタックな環境を試す(数回経験する事を推奨)
        - FE => BE => IF => CI/CD までの一連の流れのシステムを構築する経験は重要なものになる
            - そのような経験がある事でT字型の人材へ飛躍する1つの要因になる
