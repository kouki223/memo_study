# プロキシ

プロキシとは、代理を意味する言葉で、通常、プロキシと称されるものは厳密には、フォワードプロキシと呼ばれるものになる。

## Use Case

特定の企業において、ネットワーク通信を秘匿する役割を持っている。
クライアントのIP => インターネット => Webサーバーではなく、プロキシのIP => インターネット => Webサーバーという流れになる(フォワードプロキシ)

リバースプロキシにおいては、インターネット => Webサーバーではなく、インターネット => リバースプロキシ => Webサーバーという形にするしている

## プロキシの種類と役割

種類
1. フォワードプロキシ => 利用者の外向きの通信を集約して管理する
    - 内部ネットワークを秘匿する役割を持っている
        - 実際にやっている事
            - キャッシュ
            - フィルタリング
            - 匿名性(セキュリティ)
            - ログ/監査

2. リバースプロキシ => Webサービスへのアクセスを守りながら振り分けていく
    - Webサーバーの代理の役割を持つ
        - 目的としては、クライアントが直接Webサーバーに触る事ができないようにしている
            - 実際にやっている事
                - 負荷分散
                    - Webサーバーは基本複数台のインスタンスを置き、冗長化している想定
                        - それぞれのインスタンスへ負荷を分散している
                - セキュリティの向上
                    - Webサーバーを隠す
                - SSLの終端
                    ![ネットワーク通信のプロトコル図解](../../../Public/ネットワーク通信プロトコル図解.webp)
                    - HTTPS通信のパケットにおいて
                        - TCP 3Wayハンドシェイク => TLS通信 => http通信 という流れをとる
                            - TLS通信部分を担当する
                                - Webサーバーがこの暗号化ちお複合化をしなくてよいように通信へ専念させる
                - キャッシュ
        - ロードバランサー
            - 時にはリバースプロキシを代用する可能性がある

### 通信の流れ

社内からインターネット経由で Web サーバーへ届くまでの一例。フォワードプロキシは**クライアント側（外向き通信の出口付近）**、リバースプロキシは**サーバー側（インターネットから見た手前）**に置く想定。

```
クライアント
│
├ フォワードプロキシ
│
├ スイッチ
│
├ ファイアウォール
│
├ ルーター
│
├ インターネット
│
├ リバースプロキシ
│
└ Webサーバー
```
