# OSSとは

- (opensource intiative)[https://opensource.org/license]
    - OSSと定義できるもの
        - ライセンスを確認できる
            - BSDライセンス
            - BSD LIKE ライセンス => OSSとは呼ぶことが出来ない

- フィードバック
    - upstreamに報告すること
        - バグ報告
        - 新規機能要望

- 報告時に意識する3点
    - 作業手順
        - 誰がどのように読んでも一意になることを目的に記述する
    - 期待される結果
        - 場合によっては期待している事になる可能性もあり得る
    - 実際の結果

- 記述する時に重要な点
    - OSのバージョン
    - ライブラリのバージョン
    - 作業までの手順
        - これらの細かな情報を適切に記述する

- 想像について
    - 想像している事に違いが多く出てくる事がある
        - 開発者(upstream側の想像している事と報告者側が想像している事の違い)
            - なので、適切な情報を適切に余す事なく記述する

- OSS gateの作業ログやなどなど
    - https://github.com/oss-gate/workshop/issues
        - ここに過去分がすべて載っているので参考にする
