# ログポイズニング

## Webシェル設置型：SQLログインジェクション
例：
phpMyAdmin経由でSQLを実行する権限をつける
↓
SQLクエリ画面から複数のSQLコマンドを実行
MySQLの 一般クエリログ（general query log） を有効化
ログの出力先ファイルを、Webからアクセスできるディレクトリかつ拡張子を .php に設定
実行したSQL文の中にPHPコードを混ぜる → そのコードが「ログ」としてphpファイルに書き込まれる
そのphpファイルにブラウザ（POSTリクエスト）でアクセスすると、書き込まれたPHPコードがサーバー上で実行される＝Webシェル完成
これがいわゆる「ログポイズニング（Log Poisoning / Log Injection）」という古典的かつ強力なWebシェル設置手法です（「Logインジェクション技術」がこれ）。
